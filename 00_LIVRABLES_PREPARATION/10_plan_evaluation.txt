RÈGLE : Sans ça, on te dira "c'est subjectif".

================================================================================

OBJECTIFS DE L'ÉVALUATION
================================================================================

1. Valider la PRÉCISION du modèle d'estimation
2. Mesurer l'UTILITÉ pour les utilisateurs
3. Identifier les LIMITES et axes d'amélioration
4. Prouver la SUPÉRIORITÉ vs estimation intuitive


================================================================================

MÉTRIQUES DE PRÉCISION
================================================================================

MÉTRIQUE 1 : Erreur d'Effort
-----------------------------

Formule :
Erreur_Effort(%) = |Effort_Estimé - Effort_Réel| / Effort_Réel × 100

Pour chaque cas de test :
• Calculer erreur individuelle
• Agréger : erreur médiane, erreur moyenne, écart-type

Objectifs :
□ Erreur médiane ≤ _____% 
□ Erreur moyenne ≤ _____% 
□ 75% des cas avec erreur < _____% 

Référence acceptable (littérature) :
• Excellent : < 25%
• Bon : 25-40%
• Acceptable : 40-60%
• Faible : > 60%


---

MÉTRIQUE 2 : Erreur de Durée
-----------------------------

Formule :
Erreur_Durée(%) = |Durée_Estimée - Durée_Réelle| / Durée_Réelle × 100

Objectifs :
□ Erreur médiane ≤ _____% 
□ Erreur moyenne ≤ _____% 


---

MÉTRIQUE 3 : Couverture de l'Intervalle
----------------------------------------

Formule :
Couverture(%) = Nb_Cas_Dans_Intervalle / Nb_Cas_Total × 100

Un cas est "dans l'intervalle" si :
Effort_Réel ∈ [Effort_Bas, Effort_Haut]

Objectifs :
□ Couverture ≥ 70%
□ Idéal : 80-85%

Si < 70% → intervalle trop étroit (sous-estimation de l'incertitude)
Si > 95% → intervalle trop large (peu utile)


---

MÉTRIQUE 4 : Distribution des Erreurs
--------------------------------------

Analyser si les erreurs sont :
• Symétriques (autant de sur-estimations que de sous-estimations)
• Biaisées (tendance à sur/sous-estimer systématiquement)

Biais = Moyenne(Effort_Estimé - Effort_Réel)

Objectif :
□ |Biais| < 10% (erreurs compensées)

MÉTRIQUES D'UTILITÉ
================================================================================

MÉTRIQUE 5 : Satisfaction Utilisateur
--------------------------------------

Questionnaire simple (échelle 1-5) :

1. L'estimation est-elle crédible ?
   1 = Pas du tout ... 5 = Totalement
   
2. L'estimation vous aide-t-elle à décider ?
   1 = Pas du tout ... 5 = Énormément
   
3. Les explications sont-elles claires ?
   1 = Confuses ... 5 = Très claires
   
4. Utiliseriez-vous cet outil régulièrement ?
   1 = Jamais ... 5 = Systématiquement
   
5. Note globale de l'outil ?
   1 = Inutile ... 5 = Indispensable

Score moyen visé : ≥ 3.5/5

Nombre d'utilisateurs testeurs : _____ (minimum 5-10)


---

MÉTRIQUE 6 : Amélioration vs Intuition
---------------------------------------

Comparer :
• Estimation de l'outil
• Estimation intuitive (expert humain sans outil)

Sur même ensemble de cas.

Objectif :
□ Erreur de l'outil < Erreur humaine (au moins pour 70% des cas)


================================================================================

PROTOCOLE D'ÉVALUATION
================================================================================

PHASE 1 : Calibration
----------------------

Dataset : Utiliser _____ cas du Truth Dataset (Livrable 7)

Méthode :
1. Ajuster les paramètres du modèle (a, b, multiplicateurs)
2. Minimiser l'erreur sur cet ensemble
3. Documenter les paramètres finaux

Résultat :
• Modèle calibré v1.0
• Paramètres optimisés documentés


---

PHASE 2 : Test (Validation)
----------------------------

Dataset : Réserver _____ cas NON utilisés pour calibration

Méthode :
1. Appliquer le modèle calibré (sans modification)
2. Comparer estimations vs réalité
3. Calculer métriques 1-4

Si dataset < 20 cas total :
→ Utiliser validation croisée (leave-one-out)
   = calibrer sur N-1 cas, tester sur le Nème, répéter


Résultats attendus :
┌─────────────────────────┬──────────┬────────────┐
│ Métrique                │ Valeur   │ Objectif   │
├─────────────────────────┼──────────┼────────────┤
│ Erreur médiane effort   │ ____%    │ ≤ ____%    │
│ Erreur médiane durée    │ ____%    │ ≤ ____%    │
│ Couverture intervalle   │ ____%    │ ≥ 70%      │
│ Biais                   │ ____%    │ < 10%      │
└─────────────────────────┴──────────┴────────────┘


---

PHASE 3 : Test Utilisateur
---------------------------

Participants : _____ utilisateurs cibles
(chefs de projet, responsables administratifs...)

Protocole :
1. Présenter l'outil (5 min)
2. Demander d'estimer 2-3 projets qu'ils connaissent
3. Comparer avec leur propre estimation intuitive
4. Questionnaire de satisfaction (métrique 5)
5. Interview feedback qualitatif (15 min)

Questions ouvertes :
• Qu'avez-vous trouvé utile ?
• Qu'est-ce qui manque ?
• Utiliseriez-vous cet outil ? Pourquoi ?
• Suggestions d'amélioration ?


Résultats attendus :
• Score satisfaction moyen : ___/5
• Feedback qualitatif consolidé
• Liste des améliorations prioritaires
